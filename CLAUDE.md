# Design Specialist Project

このプロジェクトは、**独自性のある、プロダクショングレードのフロントエンドインターフェース**を構築することを目的としています。

## プロジェクトの方針

### デザイン思想

ジェネリックな「AI生成っぽい」デザインを避け、明確な美的方向性を持った、記憶に残るUIを実装します。

**重要な原則**: 明確なコンセプトの方向性を選び、精密に実行すること。大胆な最大主義も洗練された最小主義も機能する—鍵は、強度ではなく**意図性**にあります。

### デザイン思考プロセス

コードを書く前に、以下の4つの項目を検討し、**大胆な美的方向性**にコミットします：

#### 1. Purpose（目的）
- このインターフェースはどのような問題を解決するのか？
- 誰が使用するのか？

#### 2. Tone（トーン）
極端な美的方向性を選ぶ：
- 極端なミニマリズム
- 最大主義的な混沌
- レトロフューチャー
- オーガニック・自然的
- ラグジュアリー・洗練
- 遊び心・おもちゃ的
- エディトリアル・雑誌風
- ブルータリスト・生々しい
- アールデコ・幾何学的
- 柔らかい・パステル調
- 工業的・実用主義的

#### 3. Constraints（制約）
- 技術要件（Astro、パフォーマンス、アクセシビリティ）
- デバイス対応、ブラウザサポート

#### 4. Differentiation（差別化）
- このデザインを**忘れられないもの**にする要素は何か？
- ユーザーが覚えている**1つの要素**は何か？

## 技術スタック

- **Astro** - 高速な静的サイトジェネレーター
- **HTML/CSS/JavaScript** - ピュアなWeb技術
- モダンCSS（カスタムプロパティ、グリッド、フレックスボックス、アニメーション）

## フロントエンド美学ガイドライン

### 1. タイポグラフィ

- **美しく、ユニークで、興味深いフォント**を選ぶ
- ジェネリックなフォント（Arial、Inter、Roboto）を避ける
- 独特な**ディスプレイフォント**と洗練された**本文フォント**をペアリング
- Google Fontsや他のフォントサービスを活用

### 2. カラー & テーマ

- **一貫した美的方向性**にコミット
- CSS変数を使用してテーマを管理
- **支配的な色と鋭いアクセントカラー**は、控えめで均等に分散されたパレットを上回る
- 大胆な色使いを恐れない

### 3. モーション & アニメーション

- エフェクトとマイクロインタラクションにアニメーションを使用
- **CSSのみのソリューション**を優先
- **戦略**: ページロード時の1つの調整された演出 > 散在するマイクロインタラクション
- **高いインパクトのある瞬間**に集中：
  - 時間差のある表示（animation-delay）を使った、よく調整されたページロード演出
  - スクロールトリガーと驚きのあるホバーステート
  - IntersectionObserverを活用したスクロールアニメーション

### 4. 空間構成

- 予想外のレイアウト
- 非対称性と重なり
- 斜めの流れ
- グリッドを破る要素
- 豊かなネガティブスペース **OR** コントロールされた密度

### 5. 背景 & ビジュアルディテール

単色ではなく**雰囲気と深み**を作る。美的方向性に合わせたコンテキストに応じたエフェクトとテクスチャを追加：

- グラデーションメッシュ
- ノイズテクスチャ
- 幾何学パターン
- レイヤー化された透明度
- ドラマチックなシャドウ
- 装飾的なボーダー
- グレインオーバーレイ
- グラスモーフィズム（backdrop-filter）

## 避けるべきもの（アンチパターン）

**ジェネリックなAI生成の美学を決して使用しない：**

❌ 使い古されたフォントファミリー（Inter、Roboto、Arial、システムフォント）
❌ 陳腐なカラースキーム（特に白背景上の紫グラデーション）
❌ 予測可能なレイアウトとコンポーネントパターン
❌ コンテキスト固有の個性に欠けるクッキーカッターデザイン

## 実装の基準

- プロダクショングレードで機能的
- 視覚的に印象的で記憶に残る
- 明確な美的観点で一貫性がある
- すべてのディテールが細心の注意を払って洗練されている
- レスポンシブでアクセシブル
- パフォーマンスが最適化されている

## デザインレビューのチェックリスト

デザインをレビューする際は、以下を確認：

1. **タイポグラフィのバランス** - フォントサイズ、行間、字間が適切か
2. **カラーコントラスト** - アクセシビリティ基準（WCAG AA以上）を満たすか
3. **アニメーションのタイミング** - スムーズで自然な動きか
4. **レスポンシブデザイン** - モバイル、タブレット、デスクトップで適切に表示されるか
5. **細部のディテール** - ボーダー、シャドウ、余白などが洗練されているか
6. **独自性** - 他と差別化された、記憶に残る要素があるか

## ワークフロー

### デザイン実装時

1. **コンテキストを理解** - 目的、ユーザー、ブランドを把握
2. **美的方向性を決定** - 極端な方向性を選び、コミットする
3. **実装** - 選んだ方向性を精密に実行
4. **プレビュー** - ブラウザで実際の表示を確認
5. **洗練** - 細部を磨き上げる

### デザインレビュー時

1. **開発サーバーを起動** - `npm run dev`
2. **ブラウザで確認** - 実際の表示をチェック
3. **チェックリストで評価** - 上記のレビューポイントを確認
4. **改善を実装** - 必要な調整を行う
5. **再確認** - 変更が意図通りか検証

## 重要な原則

### 1. 創造的な解釈

**創造的に解釈**し、コンテキストに真に合わせてデザインされたと感じられる予想外の選択をする

### 2. 多様性の重視

**同じデザインは作らない** — プロジェクトごとに以下を変化させる：
- ライト/ダークテーマの選択
- 異なるフォントファミリー
- 異なる美的方向性
- 異なるカラーパレット

### 3. 収束を避ける

世代を超えて**決して一般的な選択に収束しない**

**具体的に避けるべき「トレンド」フォント**:
- Space Grotesk
- Instrument Sans
- その他の流行りのフォント

常に新鮮で独自の選択を心がける。

### 4. 実装の複雑さをビジョンに合わせる

**重要**: 実装の複雑さを美的ビジョンに一致させる

- **マキシマリストデザイン**:
  - 広範なアニメーションと効果を備えた精密なコード
  - レイヤー化された視覚効果
  - 複雑なインタラクション

- **ミニマリスト/洗練されたデザイン**:
  - 制約と精密性
  - スペーシングとタイポグラフィへの細心の注意
  - 微妙なディテールへの配慮
  - 少ない要素で最大の効果

### 5. エレガンスはビジョンの実行から生まれる

**エレガンス**は、ビジョンをいかにうまく実行したかから生まれる
